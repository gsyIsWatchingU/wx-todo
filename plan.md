# 统一页面视觉优化方案

## Summary
目标是把当前小程序的 `auth`、`today`、`task-edit` 和 `index` 页面统一到一套“简洁、偏可爱、轻涂鸦”的视觉语言里，同时统一页面边距、卡片层级、按钮样式和空状态表现。
这次按“允许小幅改结构”执行，但只做轻量布局调整，不改业务流程、不改交互逻辑、不碰登录和任务数据流本身。

## Key Changes
### 1. 建立全局可爱涂鸦主题
在全局样式层新增一套统一 token，替换现在偏默认业务风的变量：
- 主色改为柔和暖色系/薄荷系的低饱和组合，避免现在登录页渐变和其他页面割裂。
- 补充页面外边距、卡片内边距、圆角、描边、阴影、标签、按钮高度等统一尺寸变量。
- 增加“手绘感”基础样式：轻描边、不规则阴影、贴纸感标签、柔和背景纹理/渐变，但控制在轻涂鸦强度，不做重装饰。
- 在 [src/app.scss](/E:/new-study/wx-todo/src/app.scss) 和样式基础文件中补一层通用页面骨架类，例如页面容器、内容区、卡片、胶囊按钮、空状态容器。

### 2. 统一页面骨架与边距
对主要页面统一外层结构和间距规则：
- 所有页面主内容区统一使用同一水平边距和分段间距，避免 `today`、`task-edit`、`index`、`auth` 各自一套。
- 顶部标题区统一为“浅底卡片/浮层 header + 内容区卡片”的结构，保证页面切换时风格连续。
- 浮动按钮、底部 quick add、表单区等统一圆角、描边和阴影逻辑，减少当前视觉断层。

### 3. 页面级视觉重做
按页面分组实施：
- 登录页：保留现有头像选择和昵称输入逻辑，只重做为“欢迎卡片 + 贴纸感插画/涂鸦点缀 + 柔和主按钮 + 更统一输入框”的样式；不覆盖你当前 `auth` 业务改动。
- 今日页：统一顶部切换器、导航按钮、quick add、FAB、日/周/月/列表视图容器，让它们都落在同一套涂鸦卡片语言里。
- 任务编辑页：把表单 section、优先级按钮、日期时间 picker、删除区统一成同系列卡片与胶囊控件。
- 首页 `index`：从现在的 demo 风格改成和主流程一致的页面；若它仍保留在路由里，就按同一主题完成，不单独走另一套视觉。

### 4. 组件级统一
对复用组件和子视图补统一规范：
- `TaskItem` 改成统一任务卡片风格，复选框、优先级点、时间标签都与主主题一致。
- `DayView`、`WeekView`、`MonthView`、`ListView` 的标题、筛选器、日历格子、空状态、滚动区 padding 全部对齐统一 spacing system。
- 空状态统一为同一套文案容器和装饰样式，不再出现有的页面是白卡片、有的页面只是纯文本占位。

## Public Interfaces / Constraints
- 不修改页面路由、任务接口、登录接口、数据类型定义。
- 不新增必须依赖的第三方 UI 库或字体资源，默认仅用现有 `scss` 和项目内组件完成。
- 对 [src/pages/auth/index.tsx](/E:/new-study/wx-todo/src/pages/auth/index.tsx) 与 [src/pages/auth/index.scss](/E:/new-study/wx-todo/src/pages/auth/index.scss) 采取“保留现有功能逻辑、只做结构微调与样式重做”的策略，因为这两个文件目前已有未提交修改。

## Test Plan
- 页面视觉检查：`auth`、`today` 四种视图、`task-edit`、`index` 在手机尺寸下边距、卡片圆角、按钮尺寸、标题层级一致。
- 交互回归：登录页头像选择、昵称输入、登录跳转正常；今日页视图切换、日期切换、快速添加、FAB 跳转正常；任务编辑页保存/删除/返回正常。
- 状态检查：空列表、长任务标题、已完成任务、高优先级任务、无日期任务在新样式下不溢出、不遮挡。
- 安全区检查：顶部 header 和底部 quick add/FAB 在 `safe-area-inset-top/bottom` 下显示正常。

## Assumptions
- 风格强度采用“轻涂鸦”：以简洁为主，只加入少量手绘描边、贴纸色块和可爱装饰。
- 允许小幅改结构，但仅限页面展示层级调整，不改变现有业务行为。
- 当前项目中文文案存在编码异常迹象；本次视觉优化默认不把“全量文案修复”作为主目标，除非在实现时顺手需要修正少量受影响文本。
