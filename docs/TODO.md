# 产品路线图

本文档用于描述 `wx-todo` 当前能力与后续版本规划，帮助后续开发时快速判断项目现状、优先级和扩展方向。

## 当前已具备能力

- 微信小程序端基础登录流程，基于 `wx.login` + `wechat-login` Edge Function 建立业务会话
- 任务新增、编辑、删除、完成状态切换
- 任务优先级、截止日期、截止时间、备注等基础字段
- `日 / 周 / 月 / 清单` 四种任务视图
- 清单创建、清单过滤与按清单组织任务
- `Supabase Edge Functions` 业务接口层
- 接口异常时的本地存储兜底，保证基础可用性

## 版本规划

### v1.1 高优先级增强

#### 1. 提醒能力

目标价值：
让 Todo 从“记录任务”升级为“主动提醒”，提升任务完成率和用户留存。

核心范围：
- 支持任务到期提醒
- 支持今日待办提醒
- 支持提醒开关与提醒时间配置

实现提示：
- 前端需要补充提醒配置入口、提醒状态展示和提醒设置表单
- 服务端需要补充提醒相关字段、任务提醒查询能力和定时触发方案
- 数据层需要为任务增加提醒时间、提醒开关、提醒状态等元信息

#### 2. 重复任务

目标价值：
覆盖高频周期性任务场景，例如每日打卡、每周复盘、每月缴费。

核心范围：
- 支持按天重复
- 支持按周重复
- 支持按月重复
- 基础实例生成规则仅覆盖标准周期，不支持复杂自然语言规则

实现提示：
- 前端需要补充重复规则配置与展示
- 服务端需要生成周期任务实例，或在查询时展开未来任务
- 数据层需要增加重复规则、下次生成时间、母任务与实例关系等字段

#### 3. 搜索与筛选

目标价值：
在任务数量增长后，帮助用户更快定位目标任务，避免列表失控。

核心范围：
- 按标题搜索
- 按备注搜索
- 按完成状态筛选
- 按优先级筛选
- 按清单筛选
- 按日期范围筛选

实现提示：
- 前端需要新增搜索栏、筛选面板和筛选状态回显
- 服务端任务列表接口需要支持搜索词和筛选参数
- 数据层需要为常用查询条件预留合适索引与查询结构

### v1.2 中期增强

#### 1. 数据统计

目标价值：
增强任务完成反馈，让用户能看到自己的使用成效和完成趋势。

核心范围：
- 今日完成数
- 本周完成数
- 完成率
- 按清单分布统计
- 按优先级分布统计

实现提示：
- 前端需要新增统计入口或统计卡片
- 服务端需要聚合统计接口，避免前端自行拼装复杂统计逻辑
- 数据层需要明确完成时间、统计口径和时间范围查询方式

### v1.3 离线能力增强

#### 1. 离线同步增强

目标价值：
把当前“接口失败时本地兜底”升级为“离线期间可持续操作，恢复网络后自动同步”。

核心范围：
- 离线新增任务
- 离线编辑任务
- 离线完成任务
- 本地操作入队
- 恢复网络后自动同步
- 展示最近同步状态

实现提示：
- 前端需要增加本地操作队列、同步状态提示和同步失败反馈
- 服务端需要支持幂等写入或可安全重放的同步接口
- 数据层需要增加同步元信息，例如本地版本、同步状态、最近同步时间

## 后续可选扩展

- 任务模型未来可以继续补充更细的提醒、重复规则和同步元信息
- 任务列表接口未来可以继续扩展搜索词、筛选参数和排序能力
- 本地存储策略未来可以从“异常兜底”进一步演进到“离线优先”

## 当前路线图边界

本路线图默认不包含以下方向，除非后续单独立项：

- 多人协作或共享清单
- 复杂通知中心与订阅消息全链路配置
- 全文搜索或高级筛选表达式
- 复杂冲突合并 UI，仅采用基础同步优先策略

## 说明

- 本路线图面向产品和开发协作，不展开为详细技术设计文档
- 默认优先级为：提醒能力 > 重复任务 > 搜索与筛选 > 数据统计 > 离线同步增强
- 离线同步增强基于现有本地兜底机制迭代，而不是重做整套数据层
